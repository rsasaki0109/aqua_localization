# Benchmarks

This directory stores reproducible benchmark notes for public underwater
datasets and baseline comparisons.

## Current Results

- [`fjord_1.md`](fjord_1.md): NTNU `subset-fjord/fjord_1` APE history for
  `/aqua_imu_loc/odometry` against the dataset baseline trajectory.
- [`fjord_1_yaw_frame.md`](fjord_1_yaw_frame.md): yaw-frame diagnosis for the
  same NTNU sequence.
- [`tank_aqua_slam.md`](tank_aqua_slam.md): first AQUA-SLAM Docker measurement
  on Tank Dataset `short_test`, current `aqua_localization` rows, and the RMSE
  gap to the baseline.
- [`aqua_slam_progress.md`](aqua_slam_progress.md): generated progress ladder
  showing each measured `aqua_*` Tank row's gap to AQUA-SLAM and improvement
  over the IMU+pressure+DVL anchor.
- [`aqua_slam_head_to_head.md`](aqua_slam_head_to_head.md): generated
  claim-aware head-to-head diagnosis that separates numeric wins/losses from
  blocked evidence such as diagnostic rows, short baselines, and missing
  held-out validation.
- [`aqua_slam_claim_readiness.md`](aqua_slam_claim_readiness.md): generated
  go/no-go board that combines the AQUA-SLAM claim gate with the Tank Medium
  held-out input verifier.
- [`aqua_slam_error_budget.md`](aqua_slam_error_budget.md): generated error
  budget showing the remaining RMSE gap, progress already banked, and the
  current fusion regression to fix next.
- [`mbes_beach_pond_loop_status.md`](mbes_beach_pond_loop_status.md):
  first MBES-SLAM `beach_pond` tuning measurement with loop-status counts,
  descriptor sweep output, and false-positive audit notes.

## Real-Bag Evaluation Manifest

[`real_bag_evaluation_manifest.json`](real_bag_evaluation_manifest.json) pins
the dataset cases that should become repeatable benchmark evidence before any
"beats another OSS" claim is made. The rendered run sheet lives at
[`real_bag_evaluation.md`](real_bag_evaluation.md). Regenerate it with:

```bash
ros2 run aqua_localization benchmark_manifest_report.py \
  docs/benchmarks/real_bag_evaluation_manifest.json \
  --out /tmp/aqua_real_bag_evaluation.md \
  --check-ready \
  --check-doc-artifacts
```

Use `--status measured`, `--status scaffolded`, or `--status planned` to focus
the report before a benchmark session.

## Comparison Planning

- [`oss_comparison.md`](oss_comparison.md): protocol for comparing
  `aqua_localization` against existing open-source localization and SLAM tools
  without overstating results.
- [`aqua_slam_comparison.md`](aqua_slam_comparison.md): focused comparison
  plan for SenseRoboticsLab/AQUA-SLAM, the closest current underwater SLAM
  baseline.
- [`tank_aqua_slam.md`](tank_aqua_slam.md): Tank Dataset head-to-head table and
  commands for AQUA-SLAM, including ROS 1 `/AQUA_SLAM/orb_odom` export into TUM
  format. Use `benchmark_gap_report.py` on this page to compute the current
  RMSE gap and the percentage improvement needed to tie the baseline.
- [`aqua_slam_progress.md`](aqua_slam_progress.md): generated by
  `aqua_slam_progress_report.py` when you want one table that answers whether a
  new `aqua_*` row actually moved closer to AQUA-SLAM.
- [`aqua_slam_head_to_head.md`](aqua_slam_head_to_head.md): generated by
  `aqua_slam_head_to_head_report.py` when you need the stricter answer:
  whether the best current row is a claimable win, a diagnostic-only win, a
  loss, or blocked by missing evidence. Add `--fail-without-claimable-win` to
  make that answer usable as a CI gate before public "beats AQUA-SLAM" wording.
- [`aqua_slam_claim_readiness.md`](aqua_slam_claim_readiness.md): generated by
  `aqua_slam_claim_readiness_report.py` when you need one page that shows the
  current best gap, claim blockers, Medium held-out input status, and the next
  command to unblock the comparison.
- [`aqua_slam_error_budget.md`](aqua_slam_error_budget.md): generated by
  `aqua_slam_error_budget.py`; pass the best run's visual drift and motion
  segment reports when available to split scale, extrinsic, and drift terms.
  Use `run_tank_visual_fusion_sweep.py` next to reduce the fused visual+DVL
  regression against standalone visual.

The comparison plan is intentionally conservative. It separates sensor-fusion,
visual/LiDAR SLAM, underwater simulation, and MBES replay work so each baseline
is evaluated on a fair input set.
